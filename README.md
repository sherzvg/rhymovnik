# Рифмовник - Умный поиск рифм

Веб-приложение для поиска рифм к русским словам с фильтрами по части речи, частоте использования и количеству слогов.

##  Возможности

- Поиск рифм с умным алгоритмом фонетического сравнения
- Фильтры по части речи, частоте использования и количеству слогов
- Оценка качества рифмы (scoring)
- Поддержка больших словарей (до 2.5 млн слов)
- Современный адаптивный интерфейс

## 🛠 Технологии

- **Backend**: Python 3.8+, Flask
- **Frontend**: HTML5, CSS3, Vanilla JavaScript
- **База данных**: JSON файлы

##  Установка

### 1. Клонирование/Создание проекта

Создайте структуру папок:
```
rhymovnik/
├── backend/
│   ├── app.py
│   ├── rhyme_engine.py
│   ├── requirements.txt
│   └── data/
│       └── (здесь будут словари)
├── frontend/
│   ├── index.html
│   ├── style.css
│   └── script.js
└── download_dictionary.py
```

### 2. Установка зависимостей

```bash
cd backend
pip install Flask flask-cors requests
```

Или через requirements.txt:
```bash
pip install -r requirements.txt
```

### 3. Загрузка словаря (ВАЖНО!)

В корневой папке проекта выполните:
```bash
python download_dictionary.py
```

Это скачает ~2.5 млн русских слов с GitHub и сохранит в `backend/data/russian_words.txt`

**Альтернатива:** Скачайте словарь вручную:
1. Перейдите на https://github.com/danakt/russian-words
2. Скачайте `russian.txt`
3. Поместите в `backend/data/russian_words.txt`

## Запуск

### Backend (API)

```bash
cd backend
python app.py
```

Сервер запустится на `http://localhost:5000`

Вы увидите:
```
Загружено 70 слов в базу  # Базовый словарь
Загружено 245678 новых слов из файла  # Если загрузили большой словарь
 * Running on http://0.0.0.0:5000
```

### Frontend (Интерфейс)

**Вариант 1:** Просто откройте `frontend/index.html` в браузере

**Вариант 2:** Используйте локальный сервер (рекомендуется):
```bash
cd frontend
python -m http.server 8000
```
Затем откройте `http://localhost:8000`

## 📖 Использование

1. **Введите слово** в поле поиска (например: "любовь")
2. **Нажмите "Найти"** или Enter
3. **Настройте фильтры** (опционально):
   - Часть речи - фильтрация по грамматической категории
   - Частота - избегайте банальных рифм, выбирая редкие
   - Слоги - подбор рифм нужной длины

## Как работает алгоритм?

### 1. Фонетическое сравнение
```python
# Слова сравниваются по окончаниям с учётом оглушения:
"любовь" → "бовь" (в произношении "боф")
"кровь" → "ровь" (в произношении "роф")
```

### 2. Оценка качества рифмы
За каждую совпадающую букву с конца слова начисляется 25 баллов:
```
любовь: б-о-в-ь
кровь:  р-о-в-ь
        ↑ ↑ ↑ ↑
Совпало: ь(+25) в(+25) о(+25) → 75 баллов
```

### 3. Фильтрация
Применяются фильтры по метаданным слов:
- Часть речи (определяется по окончанию)
- Частота использования
- Количество слогов (считается по гласным)

## API Endpoints

### POST `/api/rhymes`
Получить рифмы для слова

**Request:**
```json
{
  "word": "любовь",
  "partOfSpeech": "all",
  "frequency": "all",
  "syllables": "all"
}
```

**Response:**
```json
{
  "word": "любовь",
  "rhymes": [
    {
      "word": "кровь",
      "score": 75,
      "pos": "существительное",
      "freq": "high",
      "syllables": 1,
      "example": "Пример с словом \"кровь\""
    }
  ],
  "count": 1
}
```

### GET `/api/health`
Проверка работоспособности сервера

## 🔧 Структура кода

### `rhyme_engine.py` - Основная логика

**Ключевые методы:**

1. `_load_word_database()` - загружает слова из файла или создаёт базовую базу
2. `_get_phonetic_ending(word, length)` - получает фонетическое окончание
3. `_calculate_rhyme_score(word1, word2)` - вычисляет качество рифмы (0-100)
4. `find_rhymes(word, filters)` - находит и фильтрует рифмы
5. `_apply_filters(rhymes, filters)` - применяет пользовательские фильтры

**Метаданные слова:**
```python
{
    'pos': 'существительное',  # Часть речи
    'freq': 'high',             # Частота: high/medium/low
    'syllables': 2              # Количество слогов
}
```

## 🐛 Решение проблем

### Backend не запускается
```bash
# Проверьте установку Flask
pip install Flask flask-cors

# Проверьте порт 5000
netstat -an | grep 5000
```

### Словарь не загружается
```bash
# Скачайте вручную
wget https://raw.githubusercontent.com/danakt/russian-words/master/russian.txt
mv russian.txt backend/data/russian_words.txt
```

### CORS ошибки в браузере
Убедитесь что:
1. Backend запущен на порту 5000
2. В `app.py` есть `CORS(app)`
3. Frontend открыт через `http://` (не `file://`)

### Медленная работа
При большой базе (2M+ слов):
- Первый запуск долгий (создаётся кэш)
- Последующие запуски быстрые (используется `word_metadata.json`)
